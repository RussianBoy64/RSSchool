Introduction

to Javascript testing

What is a test?

A test is code that tests assumptions about how other code works.

Tests make code more robust and durable.

Jest

Delightful JavaScript testing framework with a focus on simplicity.

Benefits

  1. Simple start
  2. Easy to understand and use
  3. Good documentation
  4. Great for use with modern frameworks

Start using Jest


						npm install --save-dev jest
					
package.json
							“scripts” : {
								“test”: “jest”
					 		}

First test

					describe("When given 2 numbers", () => {
						it("returns the sum of those 2 numbers", () => {
							const result = add(10, 5)
							const expected = 15
							expect(result).toBe(expected)
						})
					})

Test anatomy

						const result = add(10, 5)
						const expected = 15
						expect(result).toEqual(expected)

The ideal test consists of all three stages.

Often - such a test even consists of 3 lines.

Matching Methods

toBe() toEqual() toContain() toContainEqual() toHaveLength() toBeNull() toBeUndefined() toBeDefined() toBeTruthy() toBeFalsy() toBeGreaterThan() and toBeGreaterThanOrEqual() toBeLessThan() and toBeLessThanOrEqual() toBeCloseTo() toMatch() toThrow() not

Benefits of tests

Tests will save more time in future, because they will play the role of documentation, a development tool, and verification of previously written code.
